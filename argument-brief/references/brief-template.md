# Argument Brief Template

確定・ドラフト出力時にこの構造で出力する。未確定は推測で埋めず `[未確定]`、確認待ちの事実は `[要検証]` と明記。全体で原則1ページ程度に収める。

```markdown
# Argument Brief

## Audience
誰に向けた資料か。役職・関心・置かれた状況を簡潔に。

## Decision
読んだ後、相手に何を承認 / 中止 / 開始 / 変更 / 優先順位付け してほしいか。行動として記述。

## Core claim
最終的に信じてほしいこと。原則1文・反論可能。優先順位 / 比較 / 選択 / 因果 / 適用条件 のいずれかを含む。

## Why now
今この判断が必要な具体的理由。

## Opposing view
この主張を採用しない合理的な意思決定者の、最も強い反対意見。

## Our response
Opposing view への直接的な回答。論点をずらさない。

## Non-goals
この資料で扱わないこと / 結論を広げない範囲 / 今回は決定しないこと。

## Evidence needed
- Existing evidence: 既にある根拠
- Additional evidence: 追加で取りに行く根拠
- Counter-evidence: 主張に不利になりうる証拠
- Assumptions to validate: 証明が難しいが前提にしていること
（各項目は Fact / Inference / Hypothesis を混同しない）

## Retraction condition
どんな観測可能な事実が判明したら Core claim を撤回・修正するか。
対象 / 指標 / 比較対象 / 期間 / 閾値 / 期待した因果が成立しない状態 を含める。
```

## ドラフト出力時の追記
- 末尾に「未確定の重要点」を箇条書きで残し、その後**1つだけ**最重要の質問をする。

## 確定出力時の追記
- 出力前に SKILL.md の「最終品質基準」を自己点検。
- 重大な不足があっても捏造せず `[未確定]` / `[要検証]` を残したまま最終版とする。
