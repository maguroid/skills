# agent-memory-ops

エージェント中立の「ハブ型」記憶システム。ハブワークスペース（オーケストレーションの
起点となるリポジトリ）内の `agent-memory/` ディレクトリを正本ストアとし、
特定のAIハーネスに依存しない形でエージェントの知識を蓄積・同期・整理する。

- **保存**: 1ファイル1知識のMarkdown＋索引 `MEMORY.md`。どのハーネス・ツールからも
  読み書きできる平文ストア。
- **同期**: ハブリポジトリのgit（既存のコミット・プッシュ・プルのフック）に乗せる。
  専用の同期機構は持たない。
- **接続**: ハーネスごとのアダプタで接続する。Claude Code はシンボリックリンクで
  組み込み自動メモリ機能（自動注入・自動書き込み）に、codex 等はハブの AGENTS.md の
  読み込み指示で接続。
- **衛生**: 定期GC（重複統合・命名正規化・索引整合・正本への圧縮・ハブ跨ぎ知識の
  昇格判定）。実行時期は `MEMORY.md` 冒頭の「最終GC」日付による自己記述方式が既定で、
  タスクシステムやスケジューラを必要としない。
- **境界**: ハブ境界＝信頼境界。知識は自動ではハブを跨がず、GCの昇格判定という
  意図的なゲートだけを通ってグローバル資産（CLAUDE.md・スキル・wiki）へ昇格する。

## 導入方法

スキルは「一度も呼ばれる前」には動けないため、インストール後に最初の一声が必要。

1. このスキルをディスカバリディレクトリ（`$HOME/.agents/skills` と
   `$HOME/.claude/skills`）にリンクする（リポジトリの `bootstrap.sh` で自動）。
2. エージェントに **「この環境に記憶システムをセットアップして」** と伝える。
   ハブの指定（または新設）→ ストア作成 → 同期フック設置 → ハーネスアダプタ接続 →
   GC時期の初期化まで、スキルの導入手順がガイドする。
3. 以降は、ハブでのセッションが自動でメモリを読み書きし、「最終GC」日付が
   30日を超えるとセッションがGCを提案する。

## 関連スキル

蓄積した知識・フィードバックをブリーフや受け入れ基準へ焼き付ける方法論は
`feedback-assetization` スキル（同リポジトリ）が定義元。GCの昇格判定は
その型の適用例であり、2つをセットで導入すると「記憶→抽出→資産化」のループが閉じる。
